Hybrid Apps

Hybrid apps are mobile apps implemented using web technologies. 

 

Key points are:

  • Several platforms can be supported with the same source code.
  • Apps are developed using HTML, CSS and JavaScript.

Cordova

  • Open Source project originally known as Phonegap.
  • Maintained by the Apache foundation.
  • Wraps the app in a native WebView.
  • Acts as a bridge to give access to native functions from the JavaScript running in the WebView.

A Brief History of Onsen UI

Started out as AngularJS directives for Adobe Topcoat, a CSS library providing components for mobile apps.

 

A very nice project that sadly is not actively developed anymore. :(

 

The Onsen UI CSS has since then been rewritten so we now have both our own custom CSS components and AngularJS directives.

 

That is the current state of Onsen UI.

AngularJS

Onsen UI components are implemented as AngularJS directives.

 

Just load the library and use "onsen" as a dependency in your app.

<ons-page>
  <ons-toolbar>
    <div class="center">Settings</div>
  </ons-toolbar>
  <ons-list>
    <ons-list-item>
      Push notifications are 
      {{ push ? 'activated' : 'not activated' }}.
      <ons-switch 
        ng-model="push" 
        modifier="list-item">
      </ons-switch>
    </ons-list-item>
 </ons-list>
</ons-page>

The Future of Onsen

Onsen 2.0

Onsen 2.0

  • Built on Web Components
  • Framework Agnostic
  • Minimalistic core
  • Not dependent upon Angular 1.x
  • Implemented in ES6

Onsen 2.0 Structure

Why Web Components?

  • A simpler, more lightweight API than Angular 1.x directives
  • Web Components play well with Angular 2.0.
  • Gives us a chance to reimplement and reinvent components in a better way, using knowledge we gained while developing Onsen 1.x.
  • We can support several frameworks with a shared core codebase.

Angular 1.x

  • Making the Onsen core not dependent on Angular 1.x does not mean that we are dropping support for Angular 1.x.
  • On the contrary we will keep supporting Angular 1.x as long as there is an active userbase. 
  • Angular 1.x users will continue to be our largest userbase for the forseeable future so we will continue trying to make the Angular 1.x experience better.

Angular 2.0

  • Onsen 2.0 will be released with support for Angular 2.0.
  • The core element and their internal logic is the same for both Onsen UI apps Angular 1.x, which will make migration easy.

Angular 2.0

<ons-navigator #myNav>
  <ons-page>
    <ons-toolbar>
      <div class="center">Home</div>
    </ons-toolbar>

    <ons-list>
      <ons-list-item 
        *ng-for="#item of items" 
        (click)="myNav.pushPage('details.html', {item: item})">
        {{ item.name }}
      </ons-list-item>
    </ons-list>
  </ons-page>
</ons-navigator>

Angular 1.x

<ons-navigator var="myNav">
  <ons-page>
    <ons-toolbar>
      <div class="center">Home</div>
    </ons-toolbar>

    <ons-list>
      <ons-list-item 
        ng-for="item in items" 
        ng-click="myNav.pushPage('details.html', {item: item})">
        {{ item.name }}
      </ons-list-item>
    </ons-list>
  </ons-page>
</ons-navigator>

Almost identical markup helpful when migrating an Onsen app from Angular 1.x to 2.0.

  • Directives for Angular 1.x are working and currently being tested.
  • Components and Directives are currently being developed for Angular 2.0.
